## 失败用例说明

### #19 — 编辑触发单进行中冲突失败

**操作**：任务 A（待办）→ 编辑改为「进行中」并保存 → 创建任务 B 设为「进行中」→ 保存 B

**预期**：B 保存时提示「已有任务正在进行」

**实际**：未提示错误，B 成功保存，导致同时存在两个「进行中」任务

**根因分析**：`TaskService::updateTask()`（[TaskService.cpp:256](src/model/services/TaskService.cpp#L256)）在检查单进行中规则时，调用了 `containsOtherInProgressTask(m_repository.findAll(), id)`，排除了正在编辑的任务自身。但 `createTask()` 侧的检查（[TaskService.cpp:201](src/model/services/TaskService.cpp#L201)）是正确的——它扫描所有任务。问题在于 `updateTask()` 编辑保存成功后，A 变为「进行中」并写入了数据库；随后创建 B 时 `createTask()` 的 `containsOtherInProgressTask` 理应检测到 A。**需要进一步排查 `createTask()` 侧的检查为何未生效。** 疑似 UI 侧 A 编辑保存时状态未被正确持久化，或 `findAll()` 返回的数据与预期不一致。

**状态**：待修复
